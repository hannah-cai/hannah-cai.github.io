Poetry Visualizer — creative coding

Python, basil.js/InDesign

This was a self-defined final project for two classes: Computational Design (p5/basil.js) and Coding for Humanists (Python, text analysis). Based on the requirements of the two final projects, I decided to make a poetry visualizer that uses computation to identify and highlight a poetic aspect such as rhyme, rhythm (meter), or assonance within poetry or song lyrics.

—

Moodboard

I first looked for examples of data vis and structures that could be used for data vis for inspiration. See the full board here.

Research

I then looked for existing examples of what I was trying to achieve. A google search shows a lot of fanmade, manually created content identifying rhyme in rap lyrics; Vox does something similar with their video rapping, deconstructed.

Sketches

Since the typical way to visualized rhymes seemed to be through simply highlighting text, I tried brainstorming different ways to visualize rhyme. I also decided that I would focus on assonance (vowel sounds) instead of just rhyme, since it gave a more complete picture of the composition of a text.

Surveys

I asked friends and classmates to fill out a worksheet to gather more inspiration for possible visual directions. At this point, I was considering visualizing meter as well as assonance, so I asked participants to try and visualize that as well.

Python

Identifying assonance with Python turned out to be a lot trickier than I expected. My overall method was something like this:
  1. import corpus (I was using a small corpus I created of a few favorite songs and poems)
  2. strip punctuation, tokenize to words, convert to lowercase
  3. look up pronunciations with the CMU pronouncing dictionary
  4. extract vowel phones from pronunciations
  5. look up syllables of each word with a hyphenation library
  6. store syllables and phones into a data file

Even though this was the best method I was able to come up with, I still ran into a lot of issues. The CMU pronouncing dictionary is a literal dictionary, so I had to manually find and fix pronounciations of missing words.

Additionally, I needed to break words into syllables, but couldn't find any packages for it, so I used a hyphenation package instead. This also resulted in a lot of errors, since hyphenation doesn't break words into their smallest syllables to preserve legibility (another thing I had to manually fix).

After spending a lot of time looking through and trying many methods of parsing meter, I decided to leave it out of my final visualization, since none of the methods I found were accurate enough to add anything meaningful.

Iteration 1

Before moving into basil.js, I tried using Python to generate basil code. This way, I wouldn't have to export a file and read it in a different program; I could simply generate a basil file. I eventually moved away this method because it didn't allow me to easily run for loops in basil, forcing me to generate an excessive number of lines of code. I also couldn't figure out how to get position data for characters using this method, so all of the spacing between syllables is wrong. However, this iteration does serve as a proof of concept that my final idea was achievable in the framework that I was using.

basil.js

Basil.js is a javascript framework that runs in Adobe InDesign, and it's the main program I used for the visualization aspect of this project. I opted for basil for the visuals for a few reasons; first, it was required for one of my final projects, and second, it generated print-ready files that I could use to make posters or a book. My method in basil:
  1. import corpus
  2. render text (title/author/body)
  3. comb through the data file, generating a color for each vowel sound
  4. referencing the data file, comb through the body, highlighting each syllable with its respective vowel’s color

Similarly with Python, I ran into many issues. Basil isn't a very robust platform; it's fairly buggy and not well-kept. I found that the function for parsing csv files was broken, so I had to regenerate data files as txt files and parse them in basil. Additionally, basil has no built in system for grabbing parts of words; I had to modify the built in functions for parsing characters and words in order to achieve what I wanted.

Digital Sketches

Once I got the basil program to work properly, I started experimenting with different visual approaches. I referenced my earlier sketches, but also tried some new things, including suggestions from my professors.

Final

I decided on this aesthetic (inspired by Lauren Dicioccio's work) in the end because it felt the most dynamic and visually interesting. While it sacrifices some legibility, this is a data visualization project, and I think that this organic approach showcases the data best.

Future Steps

My original plan was to create a book of visualizations; I might follow through on that in the future. I’d also love to expand this into a standalone website, with options for user input and interactivity. However, in order to do that, I’d have to put a lot more work into making the code more robust first (this would probably involve exploring APIs and other methods for grabbing pronunciations and syllables, changing the entire framework of my program).

Overall, even though I struggled a lot and ran into a lot of unforseen issues, I enjoyed this project and learned a lot! Visualizing poetry is a niche that hasn't really been filled yet, and I'd love to explore it more.
